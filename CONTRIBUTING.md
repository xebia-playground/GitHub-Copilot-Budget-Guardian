# Contributing Guide

Thank you for contributing to GitHub Copilot Budget Guardian.

## Getting Started

1. Fork the repository.
2. Create a feature branch from `main`.
3. Install dependencies:

```bash
npm install
```

4. Run tests:

```bash
npm test
```

## Development Guidelines

- Keep changes focused and small.
- Preserve existing behavior unless a bug fix is required.
- Add or update tests for all code changes.
- Keep documentation in sync with implementation.
- Follow existing project style and module boundaries.

## Commit and Pull Request Guidelines

- Use clear, descriptive commit messages.
- Include a concise problem statement and solution summary.
- Reference related issues where applicable.
- Ensure CI checks are passing before requesting review.

## Pull Request Checklist

- [ ] Code compiles and tests pass
- [ ] New or changed behavior is covered by tests
- [ ] Documentation updated where needed
- [ ] No secrets or sensitive data included

## Code of Conduct

By participating in this project, you agree to follow [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md).
